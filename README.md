# Code and data for the constant-y AGN corona paper

This repository contains the notebook, source code, tabulated data, and generated
figures used for the updated constant Compton-y slab-corona paper. The manuscript
source and compiled manuscript PDF are intentionally not included.

## Main entry points

- `main.ipynb`: final analysis and plotting notebook used for the paper.
- `plot_style.py`: plotting style helpers used by the notebook.

## Directory layout

- `data/`: observational samples and small calibration tables read directly by
  `main.ipynb`.
- `tables/`: LaTeX tables generated by the notebook.
- `figure/`: generated paper figures.
- `output/`: posterior samples and summary tables generated by the notebook.
- `pair_balance/`: scripts for compPSc energy-balance, ireflect albedo grids,
  and pair-balance critical compactness curves.
- `pair_balance/data/`: precomputed compPSc interpolation grids and pair-balance
  curves used by `main.ipynb`.
- `compps_conv/`: local convolution-version compPSc source code, validation
  scripts, and validation outputs.

## Python dependencies

The plotting notebook needs the standard scientific Python stack listed in
`requirements.txt`. Re-running the expensive compPSc and ireflect grid
generation additionally requires a working HEASoft/XSPEC installation because
the reflection kernels call XSPEC reflection models.

The notebook is designed to use the precomputed CSV files committed in this
repository, so reproducing the paper figures does not require re-running the
full compPSc grid scans.

## Reproducing the figures

From the repository root:

```bash
python -m pip install -r requirements.txt
jupyter nbconvert --to notebook --execute --inplace main.ipynb --ExecutePreprocessor.timeout=1200
```

This regenerates the files under `figure/` and `output/` that are produced by
the notebook cells.

## Notes

- The compTT optical depths in the notebook are converted from the tabulated
  slab half-depth convention to full slab optical depth.
- The final pair-balance curves are based on the local compPSc implementation,
  not the original XSPEC `compps` model.
- The ireflect-based effective albedo grids use a fixed ionization parameter
  `xi = 100` for the final interpolation tables.
